## 1. 서비스 개요

### 1.1 서비스명 및 컨셉

| 항목 | 내용 |
| --- | --- |
| 서비스명 | HWP/HWPX AI 문서 생성 데모 서비스 (v2) |
| 한 줄 설명 | 한글 문서(.hwp/.hwpx)를 업로드하면 AI가 내용을 분석해 공식 문서 초안을 자동 생성하는 localhost 데모 도구 |
| 서비스 유형 | 데스크탑 웹 애플리케이션 (localhost) |
| 대상 플랫폼 | Web Browser (Chrome, Edge, Safari 등) |

**서비스 컨셉 설명:**

Bizmatrixx 낸부 팀이 한글 문서 기반의 공식 문서(보고서, 제안서, 회의록, 공문 등)를 작성할 때, 기존 양식이나 참고 문서를 업로드하면 AI가 문서 구조를 파악하고 새로운 초안을 자동으로 생성해주는 도구이다. 브라우저에서 직접 HWP/HWPX 파일을 파싱하고 미리보기할 수 있으며, 최종 결과물은 한컴오피스에서 바로 열 수 있는 `.hwpx` 파일로 납품된다.

**배경 및 동기:**

- 기존 v1은 하드코딩된 예시 기반으로 동작하여 실제 업로드 문서 기반 생성이 불가능했다.
- `rhwp` 기반 WASM 파서의 등장으로 브라우저 내에서 HWP/HWPX를 직접 파싱·렌더링할 수 있게 되었다.
- Docker 없이 순수 Node.js + Python 환경으로 localhost에서 즉시 실행 가능한 경량 데모가 필요했다.
- 다양한 AI Provider(Anthropic, OpenAI, Kimi, xAI)를 지원하여 낸부 검증 환경을 유연하게 구성해야 했다.
- **v2 리팩토링**: 모듈형 아키텍처로 전환하고, 드래그 앤 드롭 업로드, 접근성(ARIA), 다중 페이지 텍스트 추출, 단계별 메타 정보 표시를 추가했다.

### 1.2 핵심 가치 제안 (Value Proposition)

| 항목 | 내용 |
| --- | --- |
| 핵심 가치 | 업로드한 한글 문서의 구조와 스타일을 유지하면서 AI가 새로운 공식 문서 초안을 수 초 내에 생성 |
| 차별점 | 브라우저 내 WASM 기반 HWP/HWPX 파싱(서버 부하 제로), 다중 AI Provider 지원, 업로드 양식 재사용, 드래그 앤 드롭, 접근성 지원 |
| 대안 대비 우위 | 수작업 복사/붙여넣기 대비 90% 이상 시간 단축, 기존 v1 대비 실제 문서 기반 생성 가능 |

**가치 제안 캔버스:**

| 구분 | 내용 |
| --- | --- |
| 사용자의 할 일 (Jobs) | 공식 한글 문서 초안을 빠르게 작성하고, 기존 양식을 재사용하여 일관된 서식 유지 |
| 사용자의 고통 (Pains) | 한글 문서 수작업 편집의 번거로움, 양식 훼손, AI 생성 결과를 한글로 옮기는 불편 |
| 사용자의 이득 (Gains) | 업로드만으로 초안 완성, 원본 양식 그대로 활용, 다양한 AI 엔진 비교 가능, 드래그 앤 드롭으로 직관적 업로드 |
| 고통 해결제 (Pain Relievers) | WASM 기반 브라우저 파싱, HWPX 템플릿 직접 재사용, 자동 섹션 매핑, 파일 해제 기능, 페이지 수/크기 메타 정보 |
| 이득 생성제 (Gain Creators) | 다이어그램 자동 삽입(flowchart/timeline/comparison), SVG 실시간 미리보기, AI Provider 설정 GUI, 단계별 진행 상태 |

### 1.3 타겟 사용자 정의

#### 1.3.1 주요 사용자 페르소나

**페르소나 1: 문서 작성 담당자**

| 항목 | 내용 |
| --- | --- |
| 이름 | 김문서 |
| 연령/직업 | 30대 / 기획팀 대리 |
| 기술 수준 | 중급 (한컴오피스 숙련, 웹 서비스 일상 사용) |
| 핵심 목표 | 기존 보고서 양식을 재사용해 신규 프로젝트 보고서 초안을 10분 이내에 완성 |
| 핵심 불편 | 기존 문서에서 내용만 바꾸는 작업이 반복적이고 서식이 자주 깨짐 |
| 사용 시나리오 | 지난 분기 보고서.hwp를 드래그 앤 드롭 → AI가 이번 분기 내용으로 초안 생성 → .hwpx 다운로드 후 한컴에서 마무리 |
| 성공 기준 | 원본 양식이 그대로 유지된 상태로 새 내용이 채워진 문서를 받는 순간 |

| 항목 | 내용 |

| 연령/직업 | 20대 후반 / AI Lab 연구원 |

| 핵심 불편 | 모델별로 별도의 테스트 환경을 구축하고 문서 포맷을 맞추는 작업이 번거로움 |
| 사용 시나리오 | Provider Settings에서 API 키 입력 → 동일 문서로 각 모델별 초안 생성 → 품질 비교 |
| 성공 기준 | 한 화면에서 4개 Provider의 생성 결과를 빠르게 교차 비교할 수 있는 순간 |

## 2. 핵심 메커니즘

### 2.1 4단계 파이프라인

```
[업로드] ──▶ [브라우저 WASM 파싱] ──▶ [AI 초안 생성] ──▶ [HWPX 납품]
   │                │                      │                  │
   ▼                ▼                      ▼                  ▼
.hwp/.hwpx      SVG 미리보기          섹션/다이어그램      .hwpx 다운로드
드래그/클릭   텍스트 추출(다중 페이지)   JSON 응답 파싱       한컴오피스 호환
```

### 2.2 핵심 기술 스택

| 영역 | 기술 | 역할 |
| --- | --- | --- |
| 프론트엔드 | React 18 + Vite | UI 렌더링, 파일 업로드, 상태 관리 |
| WASM 파서 | `@rhwp/core` | 브라우저 내 HWP/HWPX 파싱 및 SVG 렌더링 |
| 백엔드 | Express.js (Node.js ESM) | REST API, AI Provider 중계 |
| 문서 생성 | Python 3 + lxml | HWPX ZIP 언패킹, XML 텍스트 치환, 다이어그램 임베딩, 재패키징 |
| AI | Anthropic / OpenAI / Kimi / xAI API | 한국어 공식 문서 초안 생성 |

### 2.3 신규/개선 기능 (v2 리팩토링)

| 기능 | 설명 | 관련 파일 |
| --- | --- | --- |
| 드래그 앤 드롭 업로드 | 파일을 드래그해서 놓으면 즉시 업로드 | `Uploader.jsx` |
| 접근성(ARIA) | `role="button"`, `aria-label`, 키보드 Enter/Space 지원 | `Uploader.jsx` |
| 다중 페이지 텍스트 추출 | 첫 페이지뿐 아니라 2\~3페이지의 텍스트도 추가 추출 | `useRhwp.js` (`enrichAdditionalPages`) |
| 메타 정보 표시 | 파일명, 크기, 페이지 수, 문서 형식(HWP/HWPX), 처리 방식 표시 | `Uploader.jsx`, `PreviewPanel.jsx` |
| 단계별 섹션 레이블 | "1. 원본 문서 업로드", "2. 생성 조건" 등 직관적 그룹화 | `ControlPanel.jsx` |
| 파일 해제 | 업로드 후 × 버튼으로 파일 해제 가능 | `Uploader.jsx` |
| 다운로드 링크 직접 표시 | 생성 완료 후 파일명 링크를 ControlPanel에 바로 표시 | `ControlPanel.jsx` |
| ADR 문서 | Architecture Decision Record 3건 추가 | `v2/docs/adr/` |
| 자동화 스킬/훅 | 의존성 업그레이드, dev-server 재시작, smoke-test 등 | `v2/skills/`, `v2/hooks/`, `v2/tools/` |

## 3. 콘텐츠/기능 전략

### 3.1 문서 유형 지원

| 문서 유형 | 코드 | 기본 목차 구조 | 다이어그램 기본 포함 |
| --- | --- | --- | --- |
| 보고서 | `report` | 배경, 현황, 분석, 결론, 후속 조치 | 선택적 |
| 제안서 | `proposal` | 개요, 필요성, 제안 내용, 기대 효과, 일정 | 권장 |
| 회의록 | `minutes` | 개요, 참석자, 안건, 논의 내용, 결정 사항 | 없음 |
| 공문 | `gonmun` | 수신, 제목, 본문, 맺음말, 날짜 | 없음 |
| 기본 | `base` | 제목, 섹션1\~5 | 선택적 |

### 3.2 기능 우선순위

**P0 (MVP 필수)**

- HWP/HWPX 파일 업로드 (클릭 + 드래그 앤 드롭)
- AI 초안 생성 (섹션 제목 + 본문 + 다이어그램)
- HWPX 파일 납품 (다운로드)
- SVG 실시간 미리보기 (원본 / 초안 / 납품)
- 접근성 기본 지원 (ARIA 속성, 키보드 조작)

**P1 (고도화)**

- 다중 AI Provider 전환
- 업로드 HWPX 양식 템플릿 재사용
- 다이어그램 자동 삽입 (flowchart, timeline, comparison)
- 다중 페이지 텍스트 추출
- 파일 해제 및 재업로드
- OAuth 기반 인증 지원

**P2 (향후)**

- 협업 편집 (댓글, 버전 관리)
- 클라우드 저장 연동
- 배치 처리 (다수 문서 일괄 생성)

## 4. 수익화 전략

본 서비스는 **낸부 데모/검증 도구**로 기획되었으며, 단기 수익화 목표는 없다. 중장기적으로 다음 방향을 검토한다:

| 단계 | 시점 | 방향 |
| --- | --- | --- |
| Phase 1 | 2026 Q2 | 낸부 업무 효율화 도구로 운영, 사용성 데이터 축적 |
| Phase 2 | 2026 Q3 | SaaS 전환 검토 (클우드 기반, 구독형) |
| Phase 3 | 2026 Q4 | 기업용 API 상품화 (HWPX 생성 엔진 외부 제공) |

## 5. MVP 스코프 정의

### 5.1 포함 범위 (In-Scope)

- [x] 단일 HWP/HWPX 파일 업로드 (클릭 + 드래그 앤 드롭)

- [x] 브라우저 내 WASM 파싱 및 SVG 미리보기

- [x] 다중 페이지(최대 3페이지) 텍스트 추출

- [x] 4가지 AI Provider(Anthropic, OpenAI, Kimi, xAI) 지원

- [x] 5가지 문서 유형(report, proposal, minutes, gonmun, base) 지원

- [x] AI 생성 초안의 실시간 마크다운 렌더링

- [x] HWPX 파일 생성 및 다운로드

- [x] 업로드 HWPX 양식을 템플릿으로 재사용

- [x] 다이어그램(flowchart/timeline/comparison) 자동 삽입

- [x] localhost 실행 (Docker 불필요)

- [x] 파일 해제 및 재업로드 지원

- [x] 접근성 기본 지원 (ARIA, 키보드)

- [x] 생성 파일 메타 정보 표시 (페이지 수, 크기, 형식)

### 5.2 제외 범위 (Out-of-Scope)

- [ ] 사용자 계정/인증 시스템 (OAuth 콜백만 지원, 세션 없음)

- [ ] DB 기반 데이터 영속성

- [ ] 협업 실시간 편집

- [ ] 모바일 반응형 UI (데스크탑 우선)

- [ ] HWP 출력 (HWPX만 지원)

- [ ] 다국어 지원 (한국어만)

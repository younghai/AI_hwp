> 본 문서는 HWP/HWPX AI 문서 생성 데모 서비스(v2)의 낸부 운영 정책, 데이터 처리 원칙, AI 윤리 가이드라인을 정의한다.

## 1. 정책 개요

### 1.1 목적

본 정책서는 서비스의 낸부 사용 범위, 데이터 보안, AI 생성 콘텐츠의 책임 소재, API 키 관리 등 비즈니스 측면의 규칙을 명확히 하여, 데모 도구의 안전하고 효율적인 운영을 지원한다.

### 1.2 적용 범위

- 본 서비스는 **Bizmatrixx 낸부 네트워크(localhost)에서만 실행**된다.
- 외부 인터넷 공개는 별도 보안 검토 후 결정한다.
- 모든 사용자는 본 정책을 준수해야 한다.

## 2. 데이터 처리 정책

### 2.1 문서 데이터

| 항목 | 정책 내용 |
|------|-----------|
| 업로드 문서 보관 | 업로드된 `.hwp`/`.hwpx` 파일은 생성 프로세스 종료 후 즉시 삭제된다. (`fs.unlink` 자동 처리) |
| 생성 문서 보관 | `v2/generated/` 디렉터리에 임시 저장되며, 정기적으로(주 1회) 수동 정리한다. |
| 외부 전송 금지 | 업로드 문서 및 생성 결과를 외부 클라우드에 업로드하는 행위를 금지한다. |
| 민감정보 포함 문서 | 개인정보, 기밀 정보가 포함된 문서는 AI API로 전송하지 않는다. 테스트용 샘플 문서만 사용한다. |

### 2.2 AI 데이터

| 항목 | 정책 내용 |
|------|-----------|
| API 키 관리 | `.env` 파일 또는 UI 내 메모리 저장만 허용. 코드나 로그에 키가 노출되지 않도록 주의한다. |
| 프롬프트 로깅 | AI Provider로 전송되는 프롬프트는 서버 로그에 남지 않는다. 오류 발생 시에도 민감 내용 마스킹 처리. |
| 응답 데이터 | AI 응답(JSON)은 생성 직후 파싱되어 메모리에서 처리되며, 디스크에 영구 저장하지 않는다. |

## 3. AI 윤리 및 사용 정책

### 3.1 생성 콘텐츠 책임

- AI가 생성한 문서 초안은 **참고용**이며, 최종 납품 전 담당자의 검수/수정이 필수이다.
- AI 생성 내용에 대한 법적, 윤리적 책임은 **최종 사용자(검수자)**에게 있다.
- 허위 사실, 차별적 표현, 저작권 침해 가능성이 있는 내용이 생성될 수 있으므로 주의한다.

### 3.2 사용 제한

| 금지 행위 | 사유 |
|-----------|------|
| 불법 콘텐츠 생성 | AI를 이용한 허위 공문서, 위조 문서 작성 |
| 악의적 프롬프트 인젝션 | 시스템 프롬프트 탈취, 비정상적 행위 유도 시도 |
| API 키 무단 공유 | 조직 외부 인원에게 API 키 공유 시 계정 제한 |
| 과도한 API 호출 | 비정상적인 빈도(시간당 100회 초과)의 자동화 호출 |

## 4. 보안 정책

### 4.1 실행 환경

- 서버는 `127.0.0.1`에만 바인딩되며, 외부 네트워크 인터페이스로 노출되지 않는다.
- CORS는 `CLIENT_ORIGIN` 환경변수로 제어되며, 기본값은 `http://127.0.0.1:5188`이다.

### 4.2 Google OAuth 인증

- Google OAuth 2.0을 통한 사용자 인증을 지원한다.
- `GOOGLE_CLIENT_ID`가 미설정 또는 placeholder인 경우, 개발용 Mock 로그인 폼으로 자동 폭백한다.
- OAuth 콜백은 로컬 호스트로만 리다이렉트된다.
- 세션은 `httpOnly` 쿠키로 관리되며, 24h TTL 후 자동 만료된다.
- 토큰 및 client secret은 `.env` 파일에 저장되며, 버전 관리 시스템에 포함되지 않도록 `.gitignore`에 등록한다.

### 4.3 세션 정책

| 항목 | 정책 |
|------|------|
| 세션 저장소 | 메모리 기반 (서버 재시작 시 초기화) |
| 세션 TTL | 24시간 |
| 쿠키 속성 | `httpOnly`, `SameSite=Lax` |
| 로그아웃 | 즉시 세션 삭제 + 쿠키 제거 |
| 동시 세션 | 제한 없음 (데모 환경) |

### 4.4 파일 업로드 보안

| 검증 항목 | 처리 방식 |
|-----------|-----------|
| 파일 확장자 | `.hwp`, `.hwpx`만 허용 (클리언트 + 서버 이중 검증) |
| 파일 크기 | 최대 20MB (클리언트 Uploader), 서버 Multer는 3MB JSON + 20MB 파일 제한 |
| 파일명 인코딩 | RFC 5987 디코딩 후 NFC 정규화 처리 |
| MIME 타입 | Multer의 `buffer` 기반 처리로 MIME 검증은 클라이언트에서 보조 확인 |
| Magic bytes | 서버에서 `PK\x03\x04` (ZIP/HWPX 시그니처) 검증 |

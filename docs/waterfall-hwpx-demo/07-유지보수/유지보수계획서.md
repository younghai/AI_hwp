## 1. 개요

### 1.1 목적

본 문서는 v2 데모 서비스의 지속적인 운영을 위한 유지보수 활동의 범위, 주기, 절차를 정의한다.

### 1.2 유지보수 범위

| 범위 | 내용 |
|------|------|
| 예방 유지보수 | 의존성 업데이트, 보안 패치 |
| 적응 유지보수 | AI Provider API 변경 대응, 브라우저 호환성 |
| 완전 유지보수 | 버그 수정, 성능 개선 |
| 기능 유지보수 | 새 문서 유형 추가, 신규 Provider 지원 |
| 인프라 유지보수 | ADR, skills, hooks, tools 갱신 |

## 2. 유지보수 활동

### 2.1 정기 점검

| 활동 | 주기 | 담당 | 산출물 |
|------|------|------|--------|
| 의존성 버전 확인 | 월 1회 | 개발팀 | 의존성 현황表 |
| 보안 취약점 스캔 | 월 1회 | 개발팀 | `npm audit` 결과 |
| 생성 파일 정리 | 주 1회 | 운영자 | 디스크 사용량 확인 |
| API 키 유효성 확인 | 분기 1회 | 개발팀 | Provider 연결 테스트 |
| 세션 저장소 확인 | 월 1회 | 개발팀 | 메모리 누수 및 TTL 스위프 정상 동작 |
| Google OAuth 설정 검토 | 분기 1회 | 개발팀 | client_id/secret 유효성, redirect_uri 일치 |
| ADR/Skills 검토 | 분기 1회 | 개발팀 | 구식 문서 아카이브 |

### 2.2 패치 관리

| 구성요소 | 패치 방법 | 주기 |
|----------|-----------|------|
| Node.js | 공식 LTS 버전 업그레이드 | 메이저 릴리스 시 |
| npm packages | `npm update` + 테스트 | 월 1회 |
| Python packages | `pip install --upgrade` | 분기 1회 |
| `@rhwp/core` | `npm update` | 업스트림 릴리스 시 |
| `cookie-parser` | `npm update` | 보안 패치 시 |
| hooks/tools | 스크립트 직접 수정 | 필요 시 |

### 2.3 백업

| 대상 | 방법 | 주기 | 보관 위치 |
|------|------|------|-----------|
| 소스 코드 | Git push | 매 커밋 | Git 원격 저장소 |
| `.env` 설정 | 수동 복사 | 변경 시 | 낸부 문서 관리 시스템 |
| Google OAuth credentials | `.env` 백업 | 변경 시 | 낸부 문서 관리 시스템 |
| 템플릿 파일 | Git 포함 | 매 커밋 | Git 원격 저장소 |
| ADR/Skills | Git 포함 | 매 커밋 | Git 원격 저장소 |

## 3. 변경 관리

### 3.1 변경 요청 흐름

```
변경 요청 ──▶ 영향도 분석 ──▶ 승인 ──▶ 구현 ──▶ 테스트 ──▶ 배포
```

### 3.2 버전 관리 정책

| 버전 변경 | 조건 | 예시 |
|-----------|------|------|
| 메이저 (v2 → v3) | 아키텍처 변경, 호환성 깨짐 | - |
| 마이너 (v2.1 → v2.2) | 새 Provider, 새 문서 유형 추가, Uploader 신규 기능 | v2.1: 드래그 앤 드롭, 접근성 개선 |
| 패치 (v2.1.0 → v2.1.1) | 버그 수정, 보안 패치 | v2.1.1: AI 재시도 로직 버그 수정 |

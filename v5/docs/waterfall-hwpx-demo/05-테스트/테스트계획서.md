## 1. 개요

### 1.1 목적

본 문서는 v2 데모 서비스의 품질 보증을 위한 테스트 전략, 범위, 일정, 자원을 정의한다.

### 1.2 테스트 범위

| 범위 | 포함 | 제외 |
|------|------|------|
| 기능 테스트 | 파일 업로드, AI 초안 생성, HWPX 납품, Provider 설정 | OAuth 전체 흐름 (mock으로 대체) |
| 통합 테스트 | Client-Server-Python 흐름 | 외부 AI Provider 서비스 품질 |
| 성능 테스트 | WASM 파싱, AI 응답, HWPX 생성 소요 시간 | 동시 사용자 부하 테스트 |
| 보안 테스트 | 파일 업로드 검증, CORS, localhost 바인딩 | 침투 테스트 |
| 호환성 테스트 | Chrome, Safari, Firefox 최신 버전 | 모바일 브라우저 |
| 접근성 테스트 | ARIA 속성, 키보드 조작, 스크린 리더 | 전문 접근성 감사 (WCAG 외주) |

## 2. 테스트 전략

### 2.1 테스트 단계

```
단위 테스트 ──▶ 통합 테스트 ──▶ 시스템 테스트 ──▶ 인수 테스트
(모듈별)       (흐름별)        (E2E)            (낸부 시연)
```

### 2.2 테스트 유형별 접근

| 테스트 유형 | 방법 | 도구 | 담당 |
|-------------|------|------|------|
| 단위 테스트 | 각 함수/Hook의 독립적 검증 | Vitest, React Testing Library | 개발자 |
| 통합 테스트 | API 엔드포인트 + Python 빌드 | supertest, 임시 파일 검증 | 개발자 |
| E2E 테스트 | 실제 브라우저에서 전체 흐름 | Playwright | QA |
| 성능 테스트 | 소요 시간 측정 | console.time, stopwatch | 개발자 |
| 시각적 테스트 | SVG 렌더링 결과 확인 | 눈 검증, 스크린샷 비교 | QA |
| 접근성 테스트 | 키보드만으로 흐름 완료, ARIA 검증 | Chrome DevTools (Lighthouse), axe-core | 개발자 |
| 인증 테스트 | OAuth/Mock 로그인, 세션, 로그아웃 | 브라우저 DevTools (Network, Application→Cookies) | 개발자 |
| Smoke 테스트 | 6단계 E2E 자동화 검증(health, providers, client, rhwp.js, wasm, build+marker) | `v2/tools/smoke-test.sh` | CI/CD |

## 4. 테스트 일정

| 단계 | 기간 | 산출물 |
|------|------|--------|
| 테스트 계획 수립 | 1일 | 테스트 계획서 |
| 테스트 케이스 작성 | 2일 | 테스트 케이스 문서 |
| 단위/통합 테스트 실행 | 3일 | 버그 리포트, 커버리지 리포트 |
| E2E 테스트 실행 | 2일 | E2E 리포트 |
| 성능/보안/접근성/인증 테스트 | 1일 | 성능 측정 결과 |
| 인수 테스트 (낸부 시연) | 1일 | 인수 테스트 결과 보고서 |

## 1. 파일 업로드 테스트

### TC-UPLOAD-001: 정상 HWP 업로드 (클릭)

| 항목 | 내용 |
|------|------|
| **목적** | `.hwp` 파일 클릭 업로드 및 파싱 검증 |
| **전제 조건** | 서비스 접속, WASM 로드 완료 |
| **입력** | `sample_report.hwp` (2MB) |
| **예상 결과** | SVG 미리보기 표시, 추출 텍스트 100자 이상, 소스 분석 모드로 전환 |
| **통과 기준** | SVG 렌더링 성공, 상태 메시지 "파싱 완료", Uploader에 파일명/크기/페이지 수/형식 표시 |

| 항목 | 내용 |

| **전제 조건** | 서비스 접속 |

## 6. 인증 테스트

### TC-AUTH-001: Mock 로그인 (client_id 미설정)

| 항목 | 내용 |
|------|------|
| **목적** | Google client_id 없을 때 Mock 로그인 폼 정상 동작 검증 |
| **전제 조건** | `server/.env`에 `GOOGLE_CLIENT_ID` 미설정 또는 placeholder |
| **입력** | `/auth/google` 접속 → 이메일 "test@test.com", 이름 "테스트" 입력 → Mock 로그인 클릭 |
| **예상 결과** | 팝업 닫힘, `postMessage` 수신, `/api/me` 응답에 `authenticated: true` |
| **통과 기준** | `user.email === 'test@test.com'`, `v2_session` 쿠키 설정됨 |

### TC-AUTH-002: 로그아웃

| 항목 | 내용 |
|------|------|
| **전제 조건** | TC-AUTH-001 완료 |
| **입력** | TopBar의 "로그아웃" 버튼 클릭 |
| **예상 결과** | `POST /api/logout` 호출, 세션 삭제, `user` 상태 null |
| **통과 기준** | `/api/me`가 `authenticated: false` 반환, `v2_session` 쿠키 제거됨 |

### TC-AUTH-003: 팝업 로그인 흐름 (버전 B)

| 항목 | 내용 |
|------|------|
| **목적** | `useAuth.loginWithPopup()`이 정상 동작하는지 검증 |
| **전제 조건** | 버전 B 실행 중 (`npm run dev`, 포트 5192) |
| **입력** | TopBar의 "Google 로그인" 버튼 클릭 |
| **예상 결과** | 팝업 열림 → `/auth/google` 로드 → Mock 폼 또는 Google OAuth → 로그인 성공 시 `postMessage` → 부모 창 `user` 상태 업데이트 |
| **통과 기준** | 팝업이 닫히고 TopBar에 이메일 표시됨 |

### TC-AUTH-004: 자동 오버레이 (버전 A)

| 항목 | 내용 |
|------|------|
| **목적** | 버전 A에서 미인증 시 LoginOverlay 자동 표시 및 로그인 후 숨김 검증 |
| **전제 조건** | 버전 A 실행 중 (`npm run dev:auto`, 포트 5193), 세션 없음 |
| **입력** | http://localhost:5193 접속 |
| **예상 결과** | 페이지 로드 시 전체 화면 `LoginOverlay` 표시 → Mock 로그인 완료 → 오버레이 사라짐 → 메인 워크스페이스 표시 |
| **통과 기준** | 오버레이가 로그인 전에는 렌더링되고, 로그인 후에는 제거됨 |

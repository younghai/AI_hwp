## 2. 클라이언트 상세 설계

### 2.1 Uploader 컴포넌트

**인터페이스**:
```javascript
function Uploader({ onFileSelect, currentFile, currentInsight }) {
  // currentInsight: { pageCount, mode: 'hwpx-template' | 'hwp-source' }
}
```

**낸부 상태**:
- `isDragging`: boolean — 드래그 중 시각적 피드백
- `rejectMessage`: string — 파일 거부 오류 메시지

**알고리즘**:
1. `handleDrop(event)`:
   - `event.preventDefault()` + `event.stopPropagation()`
   - `event.dataTransfer.files[0]` 획득
   - `isAcceptedFile()`: `.hwp` 또는 `.hwpx` 확장자 검증
   - 실패 시 `setRejectMessage()` + `role="alert"` 표시
   - 성공 시 `onFileSelect(file)` 호출
2. `handleDragOver(event)`:
   - `event.dataTransfer.dropEffect = 'copy'`
   - `setIsDragging(true)` → CSS `is-dragging` 클래스 적용
3. `handleDragLeave(event)`:
   - `event.currentTarget.contains(event.relatedTarget)` 검사로 자식 요소 이벤트 필터
   - `setIsDragging(false)`
4. `handleReset(event)`:
   - `event.stopPropagation()`으로 상위 클릭 이벤트 방지
   - `onFileSelect(null)`로 파일 해제

**접근성**:
- `role="button"`, `tabIndex={0}`, `aria-label`
- `onKeyDown`: Enter/Space 키로 파일 선택기 열기
- 해제 버튼: `aria-label="파일 해제"`
- 거부 메시지: `role="alert"`

### 2.2 useRhwp Hook

**인터페이스**:
```javascript
function useRhwp() {
  return {
    sourceInsight,      // { fileName, pageCount, previewSvg, extractedText, mode }
    parseStatus,        // string
    setParseStatus,     // (status: string) => void
    parseFile,          // (file: File) => Promise<Insight | null>
    builtPreview,       // { svgs: string[], pageCount: number, fileName: string, url: string }
    renderBuiltHwpx,    // (url: string, fileName: string) => Promise<BuiltPreview | null>
    clearBuiltPreview,  // () => void
  };
}
```

**알고리즘 - parseFile (개선)**:
1. `parseJobRef.current += 1`로 job ID 생성 (경쟁 조건 방지)
2. `ensureRhwp()`: `@rhwp/core` WASM 초기화 (캐시된 Promise 재사용)
3. `file.arrayBuffer()` → `new Uint8Array(buffer)` → `new HwpDocument(bytes)`
4. `document.pageCount()` → `document.renderPageSvg(0)`
5. `extractTextFromSvg(previewSvg)`로 첫 페이지 텍스트 추출
6. `isHwpx = file.name.toLowerCase().endsWith('.hwpx')`
7. `sourceInsight` 상태 업데이트 (fileName, pageCount, previewSvg, extractedText, mode)
8. **백그라운드**: `enrichAdditionalPages({ document, totalPages, initialText: firstPageText, jobId })`

**알고리즘 - enrichAdditionalPages (신규)**:
```javascript
async function enrichAdditionalPages({ document, totalPages, initialText, jobId }) {
  if (totalPages <= 1) return;
  const extractedPages = [initialText];
  for (let i = 1; i < Math.min(totalPages, 3); i += 1) {
    extractedPages.push(extractTextFromSvg(document.renderPageSvg(i)));
  }
  // 경쟁 조건 방지: 현재 jobId가 유효할 때만 상태 업데이트
  if (parseJobRef.current !== jobId) return;
  setSourceInsight((current) => ({
    ...current,
    extractedText: extractedPages.join('\n').trim()
  }));
}
```

**알고리즘 - renderBuiltHwpx (개선)**:
- 최대 5페이지까지 SVG 렌더링 (`maxPages = Math.min(pageCount, 5)`)
- `BUILT_INITIAL = { svgs: [], pageCount: 0, fileName: '', url: '' }`로 초기화

### 2.3 useDraft Hook

**인터페이스**:
```javascript
function useDraft({ setParseStatus }) {
  return {
    draft,              // AI 초안 객체 | null
    setDraft,
    draftLoading,       // boolean
    exportState,        // { loading: boolean, url: string, fileName: string, message: string }
    generateDraft,      // (params) => Promise<Draft | null>
    buildHwpx,          // (params) => Promise<{url, fileName} | null>
    downloadBuilt,      // () => void
  };
}
```

**개선사항**:
- `exportState`에 `loading` 플래그 추가 → ControlPanel 버튼 상태 연동
- `resetExport()`: exportState를 초기 상태로 리셋
- `generateDraft` 내에서 `!sourceFile` 검증 추가 → "먼저 HWP 또는 HWPX 문서를 업로드해 주세요."
- `buildHwpx`에서 `sourceMode`와 `sourceText`를 FormData에 추가

**알고리즘 - generateDraft (개선)**:
1. `!sourceFile` 검증 → early return
2. `setDraftLoading(true)` + `resetExport()`
3. `buildOptimisticDraft({ sourceInsight, docType, companyName, goal, notes, targetTitle })` → 임시 초안 생성
4. `setDraft(optimistic)` + `setParseStatus(...)` + `onOptimistic?.()`
5. `fetch('/api/generate-draft', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body })`
6. 응답에서 `payload.draft` 추출 (구조 변경: `payload.ok` + `payload.draft`)
7. 에러 시 `setDraft(null)` + `setParseStatus(\`AI 초안 생성 실패: ${error.message}\`)`

### 2.5 useAuth Hook

**인터페이스**:
```javascript
function useAuth() {
  return {
    user,              // { email, name, picture } | null
    loading,           // boolean (초기 세션 확인 중)
    loginWithPopup,    // () => void — /auth/google 팝업 열기
    logout,            // () => void — POST /api/logout 후 상태 초기화
  };
}
```

**알고리즘**:
1. **마운트 시**: `GET /api/me` (credentials: 'include') → `user` 상태 설정
2. **loginWithPopup()**:
   - `window.open('/auth/google', 'google-oauth', 'width=500,height=600')`
   - `window.addEventListener('message')`로 `google-auth-result` 수신
   - 성공 시 `refresh()` 호출 → `user` 상태 업데이트
3. **logout()**:
   - `POST /api/logout` (credentials: 'include')
   - `setUser(null)`

### 3.5 googleAuth.js (Google OAuth Router)

**특징**: Express Router로 분리. `isPlaceholderCredential()`으로 client_id가 placeholder인지 감지. v4에서는 `createAuthRouter`가 마운트되지 않으며 `googleAuthRouter`만 사용된다.

**엔드포인트**:
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/auth/google` | Google OAuth 시작 또는 Mock HTML 반환 |
| GET | `/auth/google/callback` | code → token → userinfo → 세션 생성 |
| GET | `/api/me` | 현재 세션 사용자 반환 |
| POST | `/api/logout` | 세션 삭제 + 쿠키 제거 |

**Mock 로그인 폼**:
- `GOOGLE_CLIENT_ID` 미설정 시 HTML 폼 반환
- 이메일/이름 입력 → POST 없이 바로 세션 생성
- 개발 환경에서 OAuth 없이 인증 흐름 테스트 가능

### 4.1 EmptyState 컴포넌트

**인터페이스**:
```javascript
function EmptyState({ onSampleSelect }) {
  // onSampleSelect: (file: File) => void
}
```

**동작**:
1. 마운트 시 `fetch('/api/samples')` → 샘플 목록 상태 저장

### 4.2 Toast / useToast

**인터페이스**:
```javascript
function useToast() {
  return {
    toasts,           // ToastItem[]
    addToast,         // ({ type, message, duration?, action? }) => void
    removeToast,      // (id: string) => void
  };
}

function ToastContainer({ toasts }) { ... }
function Toast({ id, type, message, duration, action, onRemove }) { ... }
```

**동작**:
- `addToast` 호출 시 고유 ID 생성, `toasts` 배열에 추가
- `duration` 기본값: 5000ms (info/success/warning), 8000ms (error)
- `setTimeout`으로 자동 `removeToast` 예약
- `action` 객체가 있으면 버튼 렌더링 (`label` + `onClick`)

### 4.3 ValidationPanel 컴포넌트

**인터페이스**:
```javascript
function ValidationPanel({ results }) {
```

**데이터 구조**:

```javascript
for (let attempt = 0; attempt < 2; attempt++) {
```

## 6. 오류 처리 설계

| 계층 | 오류 상황 | 처리 방식 | 사용자 메시지 |
|------|-----------|-----------|---------------|
| Client (Uploader) | 확장자 불일치 | `setRejectMessage` + `role="alert"` | "지원하지 않는 파일 형식입니다: {filename}" |
| Client (Uploader) | 드래그 중 에러 | `isDragging=false` | - |
| Client (useRhwp) | WASM 로드 실패 | console.error + 상태 메시지 | "문서 파서를 로드할 수 없습니다." |
| Client (useDraft) | 파일 미업로드 | early return | "먼저 HWP 또는 HWPX 문서를 업로드해 주세요." |
| Client (useDraft) | AI 호출 401 | alert + 설정 패널 유도 | "API 키가 설정되지 않았습니다." |

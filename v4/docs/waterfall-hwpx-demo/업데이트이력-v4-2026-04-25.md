## 1. 개요

v4는 v2/v3의 아키텍처(모듈 분리, 미리보기↔다운로드 바이트 동일성, Self-Learning 인프라)를 계승하면서, 사용자 경험과 유효성 검증을 강화하는 신규 기능을 추가한다. 주요 추가물: EmptyState(가이드+샘플), Toast(알림), ValidationPanel(HWPX 검증), 샘플 문서 API, semver caret `@rhwp/core` 채택.

### 3.3 02-시스템설계 (SAD / API / 화면 / DB)

#### 아키텍처 변경 (SAD)

```
Client (React + Vite + rhwp/core WASM)
├── components/ : TopBar / LoginOverlay / ControlPanel(+Uploader) / PreviewPanel / ProviderSettings
├── hooks/      : useRhwp / useDraft / useProviders / useAuth
└── lib/        : diagrams / helpers

Server (Node.js + Express)
├── routes/     : health / providers / draft / export / auth / googleAuth
├── services/   : ai / draft / hwpxBuilder
└── lib/        : errors / env / oauth / upload / providers-config / utils

Shared (client ↔ server 공용)
└── escape / validate / docTypes

External Worker
└── scripts/build_hwpx.py  (Python, Node에서 spawn)
```

#### API 설계서에 추가할 엔드포인트

| Method · Path | 목적 |
| --- | --- |
| `GET /auth/google` | Google OAuth 리다이렉트 시작 |
| `GET /auth/google/callback` | OAuth 콜백 → 세션 쿠키 발급 |
| `POST /auth/logout` | 세션 쿠키 무효화 |
| `GET /auth/session` | 현재 사용자 상태 조회 |

→ `API설계서.md` 의 표에 Google Auth 섹션 + `/api/samples`, `/api/samples/:id/file` 추가. Express 에 `credentials: true` CORS + `cookie-parser` 미들웨어 반영. v4에서는 `createAuthRouter` 미마운트.

#### 화면설계서

- 상단: `LoginOverlay` (autoLogin=true 시) → 백드롭 블러 + Google 로그인 카드
- TopBar 우측: `user-chip` (로그인 시) 또는 `google-login-btn` (비로그인 시) + status dot + icon button
- ControlPanel 업로더: `.uploader-empty` / `.uploader-filled` / `.is-dragging` / `.is-rejected` 4상태
- EmptyState: 파일 미업로드 시 3단계 가이드 + 샘플 카드 그리드 (`/api/samples` 기반)
- Toast: 우측 상단 fixed 영역, auto-dismiss (5s/8s)
- ValidationPanel: HWPX 생성 후 유효성 결과 카테고리별 아코디언
- PreviewPanel: 우선순위 `builtPreview > draft(HTML) > sourceInsight(원본 SVG) > empty`

#### DB/세션

- 세션: httpOnly 쿠키, `SameSite=Lax`
- `.env` → `server/.env` 로 API 키·클라이언트ID·시크릿 관리. **merge-write** 로 사용자 추가 키 보존

### 3.4 03-상세설계

- 신규 모듈 상세:
  - `EmptyState.jsx`: `onSampleSelect` 콜백, `/api/samples` fetch, Blob→File 변환
  - `useToast.js`: toast 배열 상태, auto-dismiss 타이머(5s/8s), 액션 버튼 지원
  - `Toast.jsx`: type별 색상/아이콘, 진행 바 애니메이션
  - `ValidationPanel.jsx`: `overall` 배지, 카테고리 아코디언, severity 아이콘
  - `samples.js`: 정적 샘플 메타데이터, `gonmun-basic` 파일 스트리밍
- `apply_smart_replacements` invariant (ADR-0003):
  - `len(filled_sections) == len(ai_sections)`
  - `section_i` 의 body 단락 채움 수 = AI 문장 수 (중복·pad 금지)
  - AI가 body 미제공 시 해당 섹션은 빈 상태 유지
- `_normalize_paragraph(p, text)` 계약:
  - 첫 text-only run 만 유지
  - 나머지 text-only run 전부 제거
  - `<hp:linesegarray>` 를 `textpos=0` 단일 세그먼트로 리셋
  - table-wrapper 단락은 건너뜀 (`_paragraph_has_direct_text`)
- `useRhwp` 계약:
  - `parseFile(file)` → sourceInsight 업데이트
  - `renderBuiltHwpx(url, fileName)` → builtPreview.svgs 갱신 (최대 5페이지)
  - `clearBuiltPreview()` → 상태 초기화 + wasm doc `.free()`

## 5. 신규·변경 기능 상세

### 4.1 Google OAuth (신규·v4 유지)

- 클라이언트: `useAuth()` hook이 `/auth/session` 폴링 + `/auth/google` 팝업 로그인 처리
- 서버: `routes/googleAuth.js` 가 authorize/callback/logout/session 엔드포인트 제공
- v4에서는 `createAuthRouter`(OpenAI/Kimi OAuth)가 마운트되지 않음
- `VITE_AUTO_LOGIN=true` 일 때 `LoginOverlay` 가 전체 화면 덮음. 비로그인 사용 시 Topbar 우측의 "Google 로그인" 버튼으로 팝업 수행.
- CORS `credentials: true` + `cookie-parser` 로 세션 쿠키 왕복 지원.

### 4.2 드래그앤드롭 업로더 (신규)

- 컴포넌트: `components/Uploader.jsx`
- 수용 확장자: `.hwp`, `.hwpx` (MIME/magic 은 서버에서 이중 검증)
- 상태: empty → dragging → filled → rejected
- 접근성: `role="button"`, Enter/Space 로 파일 피커 열기, `aria-label` 서술형
- 같은 파일 재업로드 가능 (`input.value = ''`)

### 4.4 Python 빌더 수정 (변경)

1. `_is_text_only_run(run)` / `_paragraph_has_direct_text(p)` — table-wrapper 단락 skip
2. `_normalize_paragraph(p, text)` — 단일 run 정규화 + lineseg 리셋
3. `apply_smart_replacements` 2-pass:
   - Pass 1: title/meta/heading/body 분류 (스타일 기반)
   - Pass 2: 인덱스 기반 AI 섹션 매핑 + normalize
4. `normalize_toc` → pad 로직 제거 (AI가 준 N 섹션 그대로 사용)
5. body ps 1:1 매핑, 남는 슬롯은 빈 상태 유지 (중복 금지)
6. `embed_diagrams` → `(ImportError, OSError)` catch, cairosvg 없으면 스킵 + 설치 안내

2. `http://127.0.0.1:5192` 접속 + 하드 리프레시

5. 샘플 카드 클릭 또는 HWP/HWPX 파일을 드래그하여 업로드 존에 drop
6. 왼쪽 폼 입력 → "초안 생성" → 미리보기 페이지 자동 표시
7. "HWPX 다운로드" → 파일 저장 → 한컴에서 열어 내용 검증
8. ValidationPanel 확인: HWPX 유효성 검증 결과 표시
9. Toast 알림 확인: 업로드/생성/다운로드 성공 메시지
10. 페이지 리로드 → AI 연결 상태(아이콘) 유지 여부 확인

## 8. 운영·유지보수 영향

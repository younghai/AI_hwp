## 1. 개요

### 1.1 검토 목적

v2 데모 서비스의 사용자 경험(UX) 품질을 평가하고, 개선점을 도출한다.

### 1.2 검토 기준

| 기준 | 설명 |
|------|------|
| 사용성 (Usability) | 작업 완료 속도, 오류 빈도 |
| 유용성 (Usefulness) | AI 생성 결과의 실무 활용도 |
| 사용 욕구 (Desirability) | UI/시각적 만족도 |
| 신뢰성 (Credibility) | 결과물의 정확성, 일관성 |
| 접근성 (Accessibility) | 키보드/스크린 리더 지원, ARIA 준수 |

---

## 2. 흐름별 UX 평가

### 2.1 파일 업로드 흐름

| 평가 항목 | 점수 (5점) | 코멘트 |
|-----------|-----------|--------|
| 드래그 앤 드롭 가시성 | 5 | Uploader 존이 명확하고, 드래그 중 색상 강조가 직관적임 |
| 클릭 업로드 편의성 | 4 | OS 기본 파일 선택기 사용 |
| 파일 해제 편의성 | 5 | × 버튼이 명확하고, 해제 후 즉시 초기화됨 |
| 메타 정보 표시 | 5 | 파일명, 크기, 페이지 수, 형식(HWPX 양식/HWP 문서)이 한눈에 들어옴 |
| 파싱 진행 표시 | 4 | "파싱 중..." 텍스트로 표시됨 |
| SVG 미리보기 품질 | 5 | 원본 문서가 선명하게 렌더링됨 |
| 접근성 | 5 | `role="button"`, `aria-label`, Enter/Space 키, `role="alert"` 모두 구현 |
| 개선建议 | | 20MB 제한인데 UI에는 "최대 20 MB"라고 표시됨. 서버와 클라이언트 제한 일치 확인 필요 |

### 2.2 AI 초안 생성 흐름

| 평가 항목 | 점수 (5점) | 코멘트 |
|-----------|-----------|--------|
| 입력 폼 직관성 | 5 | "1. 원본 문서 업로드", "2. 생성 조건" 단계 레이블로 그룹화가 명확함 |
| 생성 버튼 가시성 | 5 | ControlPanel 하단에 크게 배치, 상태에 따라 "초안 생성"/"초안 재생성"/"HWPX 생성 중..." 동적 변경 |
| 대기 시간 체감 | 4 | Optimistic Draft로 대기 중에도 콘텐츠 표시 |
| 결과물 품질 | 4 | 한국어 문법 적절, 섹션 구조 명확 |
| 다운로드 링크 직접 표시 | 5 | 생성 완료 후 ControlPanel에 바로 `result.hwpx 다운로드` 링크 표시 |
| 개선建议 | | 생성 진행률 퍼센트(%) 표시 추가 검토 |

### 2.3 HWPX 납품 흐름

| 평가 항목 | 점수 (5점) | 코멘트 |
|-----------|-----------|--------|
| 다운로드 버튼 가시성 | 5 | 생성 완료 후 활성화되어 명확함 |
| 최종 미리보기 유용성 | 5 | 한컴에서 열기 전에 SVG로 최종 확인 가능, "실제 다운로드 파일과 동일한 HWPX 바이트" 문구로 신뢰성 제고 |
| 파일명 가독성 | 4 | `{제목}-{timestamp}.hwpx` 형태 |
| meta-grid 정보 | 5 | 결과 파일명, 페이지 수, 문서 유형, 상태가 2×2 그리드로 깔끔하게 배치됨 |
| 개선建议 | | 다운로드 폴터 지정 옵션 검토 |

### 2.5 인증 UX 평가

| 평가 항목 | 점수 (5점) | 코멘트 |

### 2.6 접근성 평가

| 평가 항목 | 점수 (5점) | 코멘트 |
|-----------|-----------|--------|
| 키보드 네비게이션 | 5 | Tab으로 모든 요소 접근 가능, Enter/Space로 업로드/생성/다운로드 완료 가능 |
| ARIA 속성 | 5 | Uploader에 role/aria-label/tabIndex, 거부 메시지에 role="alert" 적용 |
| 초점 가시성 | 4 | 기본 브라우저 focus 스타일 사용, `:focus-visible` 커스텀 스타일 추가 권장 |
| 스크린 리더 | 4 | 파일명/크기/페이지 수가 텍스트로 표시되어 읽을 수 있음 |
| 개선建议 | | SVG 미리보기에 `aria-describedby` 또는 `figcaption` 추가 |

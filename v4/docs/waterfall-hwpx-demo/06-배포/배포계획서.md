## 1. 개요

### 1.1 목적

본 문서는 v2 데모 서비스를 개발자/검증자의 로컬 환경에 배포하는 절차를 정의한다.

### 1.2 배포 특성

- **localhost 전용**: 외부 서버 배포 없이 사용자 PC에서 직접 실행
- **Docker 미사용**: 순수 Node.js + Python 환경
- **npm workspaces**: client/server 통합 관리
- **자동화 인프라 포함**: ADR, skills, hooks, tools가 코드베이스에 포함됨

## 3. 배포 절차

### 3.1 사전 준비

```bash
# 1. 저장소 클론
git clone <repository-url> calendar_app
cd calendar_app

# 2. Python 의존성 설치 (선택)
pip install cairosvg

# 3. v4 디렉터리 진입
cd v4
```

### 3.2 환경 설정

```bash
# 1. server/.env 파일 생성
cp server/.env.example server/.env

# 2. .env 편집 (필수: 최소 1개 Provider API 키)
# ANTHROPIC_API_KEY=sk-ant-...
# OPENAI_API_KEY=sk-...
# KIMI_API_KEY=...
# XAI_API_KEY=...
#
# 선택: Google OAuth (없으면 Mock 로그인 폭백)
# GOOGLE_CLIENT_ID=xxxx.apps.googleusercontent.com
# GOOGLE_CLIENT_SECRET=xxxx
# OAUTH_REDIRECT_URI=http://localhost:8792/auth/google/callback
```

### 3.3 애플리케이션 설치 및 실행

```bash
# 1. 의존성 설치 (postinstall 자동 실행)
npm install
# → postinstall: bash scripts/setup-rhwp-symlink.sh || true

# 2. 개발 서버 실행 (버전 B: 수동 로그인)
npm run dev

# 또는 버전 A 실행 (자동 로그인 오버레이)
npm run dev:auto
```

### 3.4 접속 확인

| 서비스 | URL | 확인 항목 |
|--------|-----|-----------|
| Client (버전 B) | http://localhost:5192 | 파일 업로드 버튼, TopBar 로그인 버튼 |
| Client (버전 A) | http://localhost:5193 | 파일 업로드 버튼, 자동 로그인 오버레이 |
| Server Health | http://localhost:8792 / 8793 | `{"status":"ok"}` 반환 |
| Mock 로그인 | `/auth/google` (client_id 미설정 시) | 이메일/이름 입력 폼 표시 |
| 샘플 문서 목록 | `/api/samples` | `gonmun-basic` 등 샘플 메타데이터 반환 |

## 5. 롤백 절차

| 상황 | 조치 |
|------|------|
| v4 실행 실패 | `cd ..` 후 이전 버전 폴터 사용 |
| 의존성 충돌 | `rm -rf node_modules */node_modules && npm install` |
| Python 빌드 실패 | Python 3.9+ 설치 확인, `python3` 명령 사용 가능 여부 확인 |
| WASM 로드 실패 | Chrome 최신 버전 사용, 보안 소프트웨어 예외 설정 |
| OAuth 로그인 실패 | `.env`에 `GOOGLE_CLIENT_ID` 미설정 시 Mock 로그인으로 자동 폭백 |
